Analyst Studio — Kullanıcı Kılavuzu

Bu kılavuz, ekibin Analyst Studio'yu kurmasını ve günlük analiz işlerinde verimli kullanmasını sağlar. Adımlar sırayla uygulanmalıdır. Bir önceki adım tamamlanmadan sonraki çalışmaz.

1. Genel Bakış

Analyst Studio, iş analistinin elindeki bir gereksinim dokümanını (BRD veya süreç tarifi) alıp; kurumsal Confluence sayfaları, Jira görev geçmişi, Swagger/OpenAPI tanımları ve canlı uygulama gözlemi ile zenginleştirerek yapılandırılmış analizler ve geliştirme görevleri üreten bir masaüstü uygulamasıdır.

Ne yapar?

Doküman → süreç analizi → teknik analiz → HTML prototip → Jira görev hiyerarşisi.

Kim kullanır?

İş analistleri, sistem analistleri, ürün sahipleri, yazılım mimarları.

Nerede çalışır?

macOS masaüstünde. Tüm veri yerel diskte kalır; sadece Claude API çağrıları dışarı çıkar.

Nasıl AI üretir?

RAG mimarisi — her çıktı, sağlanan kurumsal kaynaklara dayandırılır; uydurma yasaktır.

İki ana akış A) Süreç → Teknik → Jira (en sık kullanılan)
B) BRD → Kapsam (BRD inceleme + revize karşılaştırması)

2. Hızlı Başlangıç ⏱ ~10 dakika

En hızlı yol — otomatik kurulum (önerilen):

  1. Klonla: git clone https://github.com/ugurcangir-design/Analysys_Agent.git brd-analyst-agent && cd brd-analyst-agent
  2. Kur: bash setup.sh — Python sürümünü kontrol eder, venv kurar, paketleri yükler, .env oluşturur (CLI modu), masaüstü ikonu üretir.
  3. Claude CLI giriş (ilk kez): npm install -g @anthropic-ai/claude-code → claude ile giriş yap.
  4. Başlat: Masaüstündeki Analyst Studio ikonuna çift tıkla (veya ./start.sh).
  5. Bir doküman yükle (Çalıştır sekmesi) → Süreç Analizi → ilk çıktı ~60 sn.

Manuel kurulum isterseniz Bölüm 3'teki adımları izleyin. Jira/Confluence için Bölüm 4.2'ye, ekipçe kullanım için Bölüm 14'e bakın.

3.5 Claude CLI kurulumu ve giriş ⏱ 3 dk

Analyst Studio AI çağrılarını Claude CLI üzerinden yapar — mevcut Claude.ai (Max/Pro) aboneliğinizi kullanır, analiz başına ek ücret yoktur.

4

Claude CLI'yı kur

npm yoksa önce Node.js kurun: brew install node

5

Claude'a giriş yap (bir kez)

Terminalde claude yazıp Enter'a basın. Tarayıcı açılır, Claude.ai hesabınızla giriş yapın. Bu oturum kalıcıdır — her analizde tekrar gerekmez.

claude
# → tarayıcıda Claude.ai girişi → "Login successful"
# Doğrula:
claude --version
6

.env dosyasını oluştur

bash setup.sh ile kurduysanız bu dosya zaten CLI modunda hazırdır. Manuel kurduysanız:

cat > .env <<'EOF'
USE_CLAUDE_CLI=true
EOF
chmod 600 .env
Özet Analiz başına ek ücret yok (abonelik kapsamı) · API anahtarı yönetmezsiniz · Tek sınır: Claude.ai planınızın saatlik/günlük hız limiti.
Güvenlik .env dosyası asla git'e commit edilmez (.gitignore'da, izin 0600).

Jira / Confluence bağlantısı arayüzden eklenecek (Adım 4.2).

Alternatif: Anthropic API key (CLI kullanmıyorsanız)
CLI yerine doğrudan API kullanmak isterseniz (token bazlı ücret): .env'e USE_CLAUDE_CLI yerine ANTHROPIC_API_KEY=sk-ant-api03-... yazın (console.anthropic.com → API Keys). Ekip kurulumunda bu yöntem önerilmez.
Ekipçe kullanım? Önerilen yöntem: her analist kendi makinesinde kurar (bu adımları takip ederek). Doğal paralellik + tam izolasyon sağlar. Ortak sunucu modeli istiyorsanız Bölüm 13.4'e bakın.

4.2 Atlassian (Jira + Confluence) bağlantısı opsiyonel

Jira'ya görev oluşturmak veya Confluence'ı RAG bağlamı olarak kullanmak istiyorsanız. Atlassian'ın OAuth 2.0 (3LO) uygulamasını bir kez oluşturup uygulamadan bağlanırsınız.

1

Atlassian'da OAuth uygulaması oluştur (yönetici, bir kez)

developer.atlassian.com/console/myapps → Create → OAuth 2.0 integration

Ayarlar:

  • Callback URL: http://localhost:5002/api/jira/callback
  • Permissions (scopes):
    • Jira: read:jira-work, write:jira-work, read:jira-user, offline_access
    • Confluence: read:confluence-space.summary, read:confluence-content.all, read:confluence-content.summary

Aldığınız Client ID ve Client Secret'i .env dosyasına ekleyin:

JIRA_CLIENT_ID=...
JIRA_CLIENT_SECRET=...
JIRA_PROJECT_KEY=PROJ                # görevleri açacağınız proje
JIRA_DEFAULT_SITE=<sitename>.atlassian.net
2

Uygulamadan OAuth ile bağlan

Uygulama → Ayarlar sekmesi → Jira / Confluence Bağlantısı → Bağlan butonu.

Tarayıcı Atlassian'a yönlenir; izinleri onayladıktan sonra geri döner ve token kaydedilir.

Confluence scope güncellemesi Daha önce sadece Jira bağlantısı yaptıysanız, Confluence scope'ları sonradan eklenmiştir → bir kez yeniden bağlanmanız gerekebilir.

4.3 Veri kaynaklarını tanımla

RAG bağlamı için Confluence space ve Jira projelerini kaydet:

  1. Referanslar sekmesine git
  2. Confluence Spaces bölümüne ilgili space key'leri ekle (ör. ARCH, ~kullanici)
  3. Jira Projects bölümüne proje key'leri ekle (ör. PROJ, TEST)

4.4 İlk senkronizasyon ⏱ 2-10 dk

1

Veri Kaynakları → Güncelle

Referanslar sekmesi üst kısımdaki Güncelle butonuna bas.

Confluence sayfaları (.md) ve Jira issue'ları (.json) yerel reference/ klasörüne çekilir.

Jira task durumu filtresi Jira projelerinden referans çekilirken Backlog, To Do ve Cancel/İptal statüsündeki task'lar dışlanır — bunlar henüz işlenmemiş veya iptal edilmiş işlerdir, analize değer katmaz. In Progress, In Review, Done ve diğer çalışılmış statüler dahil edilir. Sync log'unda kaç task'ın elendiği görünür (örn. "✓ Jira [PROJ]: 142 issue (38 elendi: Backlog/To Do/Cancel)").
Süre Space/proje büyüklüğüne göre 2-10 dakika. Tek seferlik yapılır; sonradan haftada bir / değişiklik olduğunda Güncelle tekrar basılır.

4.5 Canlı uygulama referansı opsiyonel

Gerçek çalışan uygulama ekranını analiz bağlamına almak isterseniz Bağlam Filtresi'ne URL girin:

  1. Süreç/Teknik Analiz ekranında Bağlam Filtresi'ni açın
  2. MCP Ana Uygulama URL alanına başlangıç ekranını girin
  3. Gerekirse 5 adet alt URL ekleyin; Claude MCP/Chrome bu ekranları sırayla gezer
  4. Örnek ekran olarak kullan butonu açıksa süreç analizi isterleri bu ekran yapısına göre detaylandırır
Canlı gözlem Claude MCP/Chrome erişimi varsa ekran kullanımı, form alanları, validasyonlar, mesajlar ve network servisleri kaynak olarak kullanılır. Token/cookie/authorization gibi gizli değerler analiz çıktısına taşınmaz.

6.1 Dokümanı yükle

1

Çalıştır sekmesi → Girdi Dokümanı

Süreç tarifini sürükle-bırak. Desteklenen formatlar: .pdf .docx .md .txt ve görseller (.png .jpg .jpeg .webp).

⚠ Görsel (PNG/JPG) + CLI modu uyumsuz Claude CLI modunda (varsayılan) görsel dosyalar analiz edilemez — içerikleri AI'a iletilemez. Belge görsel ise: (1) PDF/DOCX/TXT/MD olarak yükleyin, veya (2) .env'de ANTHROPIC_API_KEY tanımlayıp API moduna geçin. Yanlış kombinasyonda uygulama net hata verir (sessiz başarısızlık yok).
Tek dosya Sadece bir dosya işlenir. Birden fazla dosya yüklerseniz ilki kullanılır. Çoklu doküman desteği yol haritasındadır.

6.2 Süreç Analizi ⏱ 30-90 sn

2

Süreç Analizi Çalıştır butonuna bas

AI sırasıyla:

  • Yüklediğin dokümanı baştan sona okur

  • output/surec-analizi.md üretir

Tamamlandığında otomatik olarak Çıktılar sekmesine geçer.

İlk çağrı yavaş, sonrakiler hızlı Prompt caching ile referans bağlam 5 dakika önbelleğe alınır. Aynı projede arka arkaya çalıştırmalarda ~%90 token tasarrufu.
3

Süreç analizini incele

  4. Alternatif Akışlar (AF-XXX)
  5. Hata Akışları (EF-XXX)
  6. İş Kuralları (BR-XXX)
  7. Veri Varlıkları

Her satırda Katman sütunu olur: FE BE FE+BE veya Tek tip.

Yeniden Çalıştır = cerrahi Tüm dokümanı baştan üretmez; yalnızca belirttiğin bölümü günceller. ID'ler ve format korunur.

1

AI süreç analizinden + tüm referanslardan + (varsa) canlı uygulama gözleminden + (varsa) HTML prototipten teknik analiz üretir:

Hazır süreç analizinizle doğrudan teknik analiz Elinizde zaten hazır bir süreç analizi (.md / .txt) varsa, baştan üretmeden doğrudan teknik analiz yapabilirsiniz:
  1. Çalıştır sekmesinde süreç analizi dosyanızı (.md veya .txt) yükleyin
  2. Dosya yüklenince "Sadece Teknik Analiz" butonu otomatik görünür (normal "Süreç Analizi Çalıştır" butonunun yanında)

Not: Buton yalnızca .md/.txt dosya yüklendiğinde aktiftir (PDF/DOCX için önce tam pipeline gerekir). Uygulamada zaten üretilmiş bir süreç analizi varsa buton sekmeye girince de görünür.

6.5 Jira görev hiyerarşisi — FE/BE seçimi Yeni

Teknik analiz onaylandıktan sonra Çıktılar sekmesinde teknik-analiz.md seçili iken Hiyerarşik Task Oluştur butonu görünür.

1

AI önizleme üretir

Buton tıklandığında AI:

  • Bölüm 17 İş Kırılımı'nı okur
  • 1 Epic + 3-7 Story + 2-4 Subtask/Story hiyerarşisi önerir
  • Her öğeyi FE / BE / FE+BE / Genel olarak sınıflar

Bu adımda Jira'ya hiçbir şey yazılmaz — sadece öneri üretilir.

2

Önizleme modalında seçim yap

Açılan modalda hiyerarşiyi ağaç olarak görürsün. Her satırda:

  • Checkbox (seç / kaldır)
  • Tip etiketi: Epic / Story / Subtask
  • Katman rozeti: FE BE FE+BE
  • Başlık alanı — yerinde düzenlenebilir

Akıllı kaskad:

  • Epic'i kapatırsan tüm alt öğeler kapanır
  • Story'i kapatırsan altındaki subtask'lar kapanır
  • Bir Subtask seçersen parent Story ve Epic otomatik seçilir

Senaryolar:

DurumYapılacak
Tek task yeterli Sadece Epic'i bırak, diğerlerinin işaretini kaldır → "Devam Et" → 1 issue açılır
Tam hiyerarşi Hepsini seçili bırak → bütün ağaç Jira'da açılır
Kısmi (seçmeli) İstediğin Epic/Story/Subtask'ları seç → sadece seçilenler açılır
3

Devam Et — Jira'da Oluştur

Buton aktif olduğunda (en az bir öğe seçili) bas. AI seçtiklerini Jira'da sırayla açar; her başarılı issue'nun key'ini gösterir (örn. PROJ-123).

Proje issue type yoksa Projede Epic veya Story tipi yoksa modal uyarı verir; o öğeler Task tipi olarak açılır (fallback).

6.6 Jira görevleri triyajı Yeni

Doküman yüklemeden, mevcut bir Jira Epic/Story altındaki görevleri çekip iki gruba ayırır: hızlı işleme alınacaklar ve detaylı analiz gerekenler. Sol menü → Jira Görevleri. (Jira bağlantısı gerekir — bkz. 4.2.)

1

Görevleri çek ⏱ < 5 sn

Açık bir Epic/Story anahtarı gir (örn. MBSTRADE-1133) → Görevleri Çek. Bir-seviye-alt görevler gelir: parent, Epic Link veya Relates issue-link'leriyle bağlı olanlar (üçü birleştirilir). Bu adım anında ve 0 token: yalnızca yapısal ön-tarama (başlık/açıklamada Amaç, Değişiklik, Kabul Kriteri, dosya referansı var mı). Her kartta YAPISAL ÖN-TARAMA rozeti + neye bakıldığı yazar.

2

Görevi incele

Her kartta:

  • İçeriği gör — Jira açıklamasını düzenlemeden okuma (ADF→metin başlık/madde olarak ayrıştırılır).
  • Yorumlar (n) — varsa son 3 yorum yazar + tarih ile (görevin geçmişini anlamak için).
  • ⚠ Benzer içerik uyarısı — başka görevle %35+ örtüşüyorsa sarı şeritle bilgi; tıklanır link, o karta zıplar (aynı işi iki kez analiz etmemek için).

Uzun listede üstteki arama kutusuyla filtrele (boş grup tamamen gizlenir), grup başlığına tıklayıp katla.

3

AI ile Sınıflandır opsiyonel

Daha isabetli ayrım için AI ile Sınıflandır'a bas. AI her görevin başlık+açıklamasını okur ve içeriğe özgü gerekçeyle yeniden sınıflandırır; o kartlar AI İNCELENDİ rozeti alır. Uzun listede birkaç dakika sürebilir.

4

İşle: Formatla veya Analiz Et

Her görevde iki aksiyon:

  • Standart Formatla — mevcut içeriği 4 başlıklı standart şablona (Amaç / Değişiklikler / Referans / Kabul Kriteri) çevirir. Haiku kullanır (Sonnet'in ~%10 maliyeti, hızlı). Kapsam UYDURMAZ; bilgi yoksa "(belirtilmemiş)" yazar.
  • Teknik Analiz Et — teknik analiz motoruyla görevi detaylandırır (11 başlıklı şablon, RAG referansları dahil). İki pass: Sonnet ile teknik analiz + Haiku ile ayrı bir açık sorular pass'i.
RAG ve bağlam filtresi Teknik Analiz Et, Süreç/Teknik Analiz ekranındaki Bağlam Filtresi'ni otomatik kullanır (keyword, Jira anahtarı, Confluence sayfa adı). Sayfanın üstünde filtre aktifse mavi şeritle gösterilir; "Düzenle →" linki seni filtre kutusuna götürür. Filtre yoksa tüm referanslar dahil edilir.
5

İncele → düzenle → onayla

Çıktı tam ekran modalda açılır:

  • Çıktı sekmesi: standart format veya teknik analiz; analist üzerinde değişiklik yapabilir.
  • Açık Sorular [n] sekmesi (yalnız Teknik Analiz Et'te, varsa): Q-T-NNN formatında, Kritik→Düşük sıralı. Jira'ya YAZILMAZ, sadece okuma.
  • Esc ile veya dışına tıklayarak kapanır.

Onayla ve Jira'da Güncelle → onay sorusu çıkar; kabul edersen Jira görevi güncellenir.

Dikkat — üzerine yazar Onaylama, Jira görevinin mevcut açıklamasının tamamını yenisiyle değiştirir (geri alınamaz). Önizlemeyi gözden geçirmeden onaylama. Açık Sorular Jira'ya yazılmaz.

8. Soru Defteri Yeni

AI bir analiz ürettiğinde, eksik veya belirsiz konular için açık sorular üretir. Eski sürümde bu sorular .md çıktısında pasif duruyor, cevap gelene kadar analiz tıkanıyordu. Soru Defteri bu sorunu çözer: her sorunun durumunu kalıcı kayda alır, cevaplar geldikçe analiz iteratif olarak güncellenir.

Neden gerekti? Bir analizdeki kritik sorunun cevabı PO'dan, Jira'dan veya mimari ekipten saatler/günler sonra gelebilir. Bu sürede analiz bekleyemez — Soru Defteri ile "şu an cevap yok ama unutmayalım" demek mümkün.

8.1 Banner ve Sorular sekmesi

Bir analiz çalıştırıldıktan sonra Çıktılar sekmesinin üstünde sarı/kırmızı bir banner çıkar: 

📋 ⚠ 2 KRİTİK soru cevap bekliyor — 4 açık · 1 bekleyen · 2 varsayım    Soruları aç →

Banner'a tıkla veya üstteki 📋 Sorular sekmesine geç. Sekme başlığında dinamik rozet görürsün (kaç aktif soru var).

8.2 4 Aksiyon: her soruya ne yapacağına karar ver

Her soru kartında 5 aksiyon butonu vardır. Senaryolar:

ButonNe zaman?Sonuç
✏ Cevapla Cevap elinde, hemen yazabilirsin Inline textarea açılır; durum=cevaplandı (✓ yeşil)
🤖 Varsayım Cevap belirsiz ama mantıklı bir tahmin yeterli Textarea açılır; ⚠ VARSAYIM: etiketiyle metne işlenir, durum=varsayım (mor)
⏸ Beklet PO/mimar/Jira'dan cevap bekliyor Durum=bekleniyor (sarı); analiz devam edebilir, soru hatırlatma listesinde kalır
🗑 Atla Soru gereksiz veya bizim projemizi etkilemiyor Durum=atlandı (gri, soluk); listeden filtrelenebilir
↻ Aç Yanlış kapattın, geri al Durum=açık (kırmızı şerit)
Sırala ve filtrele Sorular önceliğe göre otomatik sıralanır (Kritik → Yüksek → Orta → Düşük). Üstteki dropdown ile filtre değiştir:
  • Aktif (varsayılan): açık + bekleyen — günlük çalışmada görmek istediklerin
  • Tüm durumlar: cevaplanmış ve atlanmışlar dahil
  • Cevaplanmış / Varsayım / Atlandı: geçmişi gözden geçirmek için

8.3 Cevapları Uygula — AI ile analizi güncelle

Cevap yazdığında veya varsayım girdiğinde, bu bilgi henüz analiz dosyasına yansımamıştır — sadece soru defterinde tutulur. "Cevapları Uygula" butonuna basınca AI etkilenen bölümleri yeniden üretir:

  1. Toolbar'daki turuncu Cevapları Uygula (N) butonu — uygulanmamış cevap/varsayım sayısı badge'inde
  2. Tıkla, onayla → AI 30-90 saniye içinde:
    • Her CEVAP için: bilgiyi ana metnin doğru yerine yerleştirir, soruyu Açık Sorular'dan kaldırır
    • Her VARSAYIM için: bilgiyi yerleştirir ama başına ⚠ VARSAYIM: etiketi ekler, soruyu varsayım durumunda bırakır
    • Diğer bölümleri DEĞİŞTİRMEZ (cerrahi refine)
  3. Sonuç toast'u: "2/2 dosya güncellendi"
  4. Sorular yeniden taranır; uygulananlar artık "uygulandı" işaretiyle gösterilir
Dikkat — geri alınamaz Cevapları uyguladıktan sonra eski .md sürümü kalmaz. Önemli bir cevap yazıyorsan iki kez kontrol edin. (Çıktının önceki hali "Geçmiş" sekmesinde olabilir, ancak garanti değildir.)

8.4 Paylaşım — bekleyenleri ekibe yolla

Toolbar'daki 📤 Paylaş butonu, açık+bekleyen soruları kopyala-yapıştır metni olarak çıkarır: 

📋 Cevap Bekleyen Sorular

• [Kritik] Q-T-001: Ödeme limiti
  Soru: Onay limiti aşımında ne olur?
  Bağlı: BR-007
  Etki: Onay endpoint'i tasarımı bekliyor

• [Orta] Q-T-002: Audit log saklama süresi
  Soru: Audit kayıtları ne kadar tutulacak?
  Bağlı: NFR-005

Modaldan 📋 Kopyala → Slack'e / e-postaya / Jira yorumuna yapıştır. Cevap geldiğinde defterden ilgili soruya ✏ Cevapla.

Kalıcılık Soru defteri output/sorular.json'da tutulur. Analizi yeniden ürettiğinde (Yeniden Çalıştır), zaten cevap verdiğin sorular KORUNUR (id + kaynak dosya eşleşmesi). AI'nın ürettiği yeni sorular "açık" olarak eklenir.

9. Çıktı Kalite Skoru Yeni

Çıktılar sekmesi açıldığında her dosyanın yanında otomatik bir kalite skoru rozeti (0-100) görünür. Rozet rengi:

  • 75-89 Sarı: İyi — küçük iyileştirmeler önerilir
  • 60-74 Turuncu: Orta — gözden geçirin

10. Üretilen Çıktıların Yapısı

📋 Yönetici Özeti (en üstte) Süreç ve teknik analiz çıktılarının başında otomatik bir özet bulunur: kapsam (kaç endpoint/tablo/bölüm), süreç kapsam yüzdesi ve açık soru sayısı (kaç kritik). 11 bölümü okumadan 10 saniyede durumu görmeniz içindir. Bu özet Jira'ya yazılmaz — yalnızca ekranda/çıktı dosyasında görünür.
⚡ Önbellek — aynı analiz neden anında geliyor? Aynı dokümanı/görevi (referans + bağlam filtresi de aynıyken) tekrar analiz ederseniz sonuç anında ve ücretsiz gelir (kaydedilen çıktı, token harcanmaz). Yeni/farklı bir sonuç istiyorsanız Yeniden Çalıştır (düzeltme notuyla) kullanın — o her zaman tazeden üretir. Bu, kullanım limitine (rate limit) takılmayı azaltır.

Tüm çıktılar Türkçe Markdown'dur. Aşamalar arası izlenebilirlik için her aşama kendi ID şemasını kullanır:

AşamaID TipleriAnlamı
BRD Analizi FR-XXX NFR-XXX US-XXX AC-XXX I-XXX Fonksiyonel gereksinim, fonksiyonel olmayan, user story, kabul kriteri, tutarsızlık
Süreç Analizi A-XXX PA-XXX BR-XXX AF-XXX EF-XXX AC-XXX EK-XXX Aktör, süreç adımı, iş kuralı, alternatif akış, hata akışı, kabul, ekran
Teknik Analiz T-FE-XX T-BE-XX FE geliştirme görevi, BE geliştirme görevi (Jira temeli)
Kapsam Analizi YE-XXX KL-XXX DG-XXX Yeni eklenen, kaldırılan, değiştirilen gereksinim
Kaynak izleme (RAG) Her somut iddia [K: kaynak] ile etiketlenir; örnek: [K: Confluence:Mimari-Karar-2024], [K: Swagger:order.json#/orders], [K: Jira:PROJ-145], [K: BRD §3.2]. Kaynaksız iddialar ana metne giremez — "Açık Sorular"a taşınır.

11.2 Düzenle vs Yeniden Çalıştır

Yeniden Çalıştır

Düzenle

Doğrudan Markdown editörü açılır. Yazım hataları, küçük metin değişiklikleri için hızlıdır. Anlam değişikliği yapacaksanız Yeniden Çalıştır daha güvenli.

"claude CLI boş yanıt döndürdü" / giriş gerekli Claude.ai oturumu yok veya süresi dolmuş. Terminal'de claude çalıştırıp Claude.ai hesabınızla giriş yapın (bir kez yeterli)
Rate limit / "abonelik sınırı" API modunda: uygulama 3 kez otomatik tekrar dener (4s, 8s, 16s). CLI modunda: Claude.ai planınızın saatlik/günlük limiti. Beklemek gerekir
Jira "401 Unauthorized" OAuth token süresi dolmuş — Ayarlar → Jira → Yeniden Bağlan
Confluence "403" OAuth scope eksik. Yeniden bağlanmak gerekir (Confluence scope'ları sonradan eklendi)
Cmd+Shift+R sonrası kapandı Düzeltildi (sürüm Mayıs 2024+). Eski sürüm kullanıyorsanız güncelleyin
Tarayıcı sekmesi kapandı, sunucu açık ~45 saniye sonra heartbeat zaman aşımıyla otomatik kapanır
Jira "Hiyerarşi Oluştur" butonu görünmüyor Sadece teknik-analiz.md seçili iken görünür. Jira bağlantısı yoksa da gizlenir
Teknik analiz başlayınca süreç analizi çıktısı kayboldu / yerine kaynak doküman geldi Eski sürüm bug'ıydı — "Sadece Teknik Analiz" butonu, input'taki .md/.txt dosyayı üretilmiş analizin üzerine kopyalıyordu. Güncel sürümde üretilmiş analiz asla ezilmez. Güncelleme → Güncelle yapın. Hazır süreç analizi yükleyerek sadece-teknik çalıştırmak isterseniz: önce Çıktılar'dan eski surec-analizi.md'yi silin, sonra dosyanızı yükleyip "Sadece Teknik Analiz"e basın
HTML Prototip üretilmiyor Önce süreç analizi olmalı. Süreç analizi yoksa veya hatalıysa prototip çıkmaz
Çıktı eksik/yanlış Yeniden Çalıştır → düzeltme notu yaz. Sürekli aynı hata → bağlam filtresi ile gürültüyü azalt veya promptu özelleştir
İndirme sırasında "Geçersiz dosya" Çıktı dosya adı izin verilen set dışında. Yeni bir çıktı tipi ekliyorsanız app.py'deki IZIN_VERILEN_CIKTILAR set'ine ekleyin
Soru defterinde sorular görünmüyor Çıktı sekmesinde toolbar'daki 🔄 Tara butonuna bas. Süreç analizi tablo formatı, teknik analiz blok formatı kullanır — ikisini de yakalar.
"Cevapları Uygula" 409 hatası Başka bir analiz çalışıyor (Çalıştır sekmesinde wf-bar görünür). Bitince tekrar deneyin.
Cross-origin POST reddedildi (403) İstek başka bir origin'den geliyor. Tarayıcı sekmesini doğru host:port'ta açtığınızdan emin olun (localhost:5002 veya kurulum IP'si).
"Admin yetkisi gerekli" (403) Bu işlem (prompt düzenleme / git pull / workflow reset) sadece ADMIN_USER için. Yöneticiniz size yetki vermeli veya admin olarak giriş yapmalı.
Uygulama başlangıçta "BAŞLANGIÇ ENGELLENDİ" diyor HOST=0.0.0.0 + AUTH_ENABLED=false kombinasyonu güvensiz. .env'de bir seçeneği değiştirin: HOST=127.0.0.1 veya AUTH_ENABLED=true veya ALLOW_LAN_NO_AUTH=true (bilinçli risk).
Login sayfası açılmıyor (AUTH_ENABLED=true ama kullanıcı yok) Kilitlendi. Terminal'den python manage_users.py ekle <username> <sifre> ile kullanıcı ekleyin.

17. Sıkça Sorulanlar

Veriler bulutta mı saklanıyor?

Hayır. Tüm dokümanlar ve çıktılar yerel makinanızda kalır. Sadece Claude API çağrıları için içerik Anthropic'e gönderilir (Anthropic veri saklama politikası geçerlidir).

Aynı dokümanı tekrar tekrar çalıştırırsam maliyet ne olur?

Bu sorunun cevabı kullandığınız moda göre değişir:

  • Claude CLI modu (önerilen, USE_CLAUDE_CLI=true): Analiz başına ek ücret yok — Claude.ai abonelik planınıza dahil. Tek sınır: planınızın hız limiti (saat/gün başına çağrı sayısı).
  • Anthropic API modu (ANTHROPIC_API_KEY kullanırsanız): Token bazlı ücretlendirme — yaklaşık tam tur 30K-50K input + 8K-16K output token. Prompt caching ile referans bağlam ve sistem promptu önbelleğe alınır → 5 dakika içindeki tekrar çağrılarda token maliyeti ~%90 düşer.

Her iki modda da gereksiz tekrar çalıştırmak yerine Yeniden Çalıştır + düzeltme notu kullanmak daha verimlidir.

Hangi modelleri kullanıyor?

claude-sonnet-4-6 tüm analizler için; claude-haiku-4-5 Jira görev başlığı üretimi gibi hafif işler için.

Birden fazla doküman aynı anda işlenebilir mi?

Şu an her çalıştırma tek doküman üzerinde. Çoklu doküman desteği yol haritasındadır.

FE ve BE görevleri nasıl ilişkileniyor?

Teknik analiz Bölüm 17'de her görev katman etiketi taşır ve "İlişkili Görev" sütununda parçası olduğu FE/BE çiftinin ID'sini gösterir (örn. T-FE-01 ↔ T-BE-02). Jira açılırken bu bilgi başlık/açıklamaya yansır; ihtiyaç olursa Jira linki manuel kurulabilir.

Promptu bozdum, nasıl geri alırım?

Ayarlar → Promptlar → ilgili prompt → Varsayılana Dön.

Çıktıyı bir başkasıyla paylaşmak istiyorum

İndir butonu ile .md dosyasını al, e-postayla paylaş. Ya da Confluence'a yayımla, link gönder.

Bu küçük "?" işaretleri ne işe yarıyor?

Arayüzdeki gri yuvarlak ? ikonları ipuçlarıdır — üstüne gelin veya odaklanın, kısa bir açıklama görünür. RAG, FE/BE katman, Kalite skoru, Bağlam Filtresi gibi kavramlar buralarda anlatılıyor. Yeni başladıysanız her ipucunu okumanız önerilir.

İlk sekmeyi açtığımda yeşil bir kutu çıktı, ne yapayım?

Bu onboarding banner'dır — o sekmenin ne işe yaradığını özetler. Okuyup × ile kapatın, bir daha gösterilmez. Tekrar görmek için tarayıcı geliştirici konsolunda _onboardSifirla() komutunu çalıştırın.

PO/Jira'dan cevap günlerce gelmezse ne yaparım?

Soru defterinin amacı tam bu durumlar için. Yapacaklar:

  1. Soruya ⏸ Beklet de — durum "bekleniyor" olur, analiz devam edebilir
  2. Mantıklı bir tahmin yapılabilirse 🤖 Varsayım kullan — bilgi metne işlenir ama ⚠ VARSAYIM etiketiyle
  3. 📤 Paylaş → bekleyenleri Slack/Jira'ya gönder, takip etmeyi unutma
  4. Bu sırada teknik analizi tamamla, Jira görevlerini aç — sonra cevap gelince ✏ Cevapla + Cevapları Uygula ile güncel hale getir

Varsayım modu güvenli mi? AI yanlış varsayım yapar mı?

Varsayım her zaman ⚠ VARSAYIM etiketiyle açıkça işaretlenir ve soru "varsayım" durumunda kalır (yeşil "cevaplandı" olmaz). Yani ekip dokümanı okurken hangi bilgilerin doğrulanması gerektiğini görür. Onay gelince ✏ Cevapla ile gerçek değerle değiştirilir.

Bir kullanıcı analiz çalıştırırken başkası ne yapabilir?

Yalnızca tek aktif analiz desteği var. Diğer ekip üyeleri: bekleyen çıktıları inceleyebilir, soru defterinde cevap yazıp kaydedebilir (ama Cevapları Uygula 409 verir), Jira/Confluence gönderim yapabilir. Eş zamanlı analiz çalıştırılamaz.

Şifremi unuttum

Yönetici sunucudaki terminalde sıfırlar: python manage_users.py sifre <username> <yeni_sifre>
