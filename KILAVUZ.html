Analyst Studio — Kurulum ve Kullanım Kılavuzu

Bu kılavuz, ekibin Analyst Studio'yu kurmasını ve günlük analiz işlerinde verimli kullanmasını sağlar. Adımlar sırasıyla uygulanmalı — bir önceki adım tamamlanmadan sonraki çalışmaz.

1. Genel Bakış

Analyst Studio, iş analistinin BRD / süreç dokümanını alıp; kurumsal bilgi kaynaklarınızla (Confluence, Jira, Swagger, UI kodu) zenginleştirerek şu çıktıları üreten bir masaüstü uygulamasıdır:

  • 📄 Süreç analizi — aktörler, adımlar, kurallar, ekranlar (FE)
  • ⚙️ Teknik analiz — DDL, API (OpenAPI YAML), validasyon, FE/BE iş kırılımı
  • ❓ Açık sorular — netleşmesi gereken konular
  • 📋 BRD analizi ve PO soruları
  • 🔄 Kapsam analizi ve alternatif süreçler
  • 🎨 HTML prototip — tıklanabilir maket
  • 🎫 Jira Epic / Story / Subtask — FE/BE etiketli, seçmeli oluşturma
  • 📘 Confluence yayımlama — analizleri kurumsal wiki'ye
İki ana akış A) Süreç dokümanı → Süreç analizi → Teknik analiz → Jira görevleri.
B) BRD → BRD analizi → (revize BRD ile) Kapsam analizi.

2.2 Depoyu klonla

1

GitHub'dan depo klonlanır

Terminal aç:

cd ~
mkdir -p projects && cd projects
git clone https://github.com/ugurcangir-design/Analysys_Agent.git brd-analyst-agent
cd brd-analyst-agent
İpucu Bundan sonra tüm komutlar ~/projects/brd-analyst-agent klasöründe çalıştırılacak.

2.3 Sanal ortam oluştur

2

Python sanal ortamını oluştur ve aktif et

python3 -m venv venv
source venv/bin/activate

Terminal başında (venv) göründüğünde ortam aktiftir.

Önemli Her yeni terminal oturumunda source venv/bin/activate komutu tekrar çalıştırılmalı. Masaüstü uygulaması (adım 2.6) bunu otomatik yapar.

2.4 Bağımlılıkları yükle

3

Pip ile paketler kurulur

pip install --upgrade pip
pip install -r requirements.txt

Yaklaşık 1-2 dakika sürer. anthropic, Flask, PyMuPDF, python-docx gibi paketler kurulur.

2.5 .env dosyası — API anahtarı

4

Anthropic API anahtarını .env dosyasına ekle

Proje kökünde .env dosyası oluştur:

cat > .env <<'EOF'
ANTHROPIC_API_KEY=sk-ant-api03-XXXX...    # kendi anahtarınız
EOF
chmod 600 .env
Güvenlik .env dosyası asla git'e commit edilmez. .gitignore tarafından zaten dışlanmıştır.
API anahtarı nasıl alınır? console.anthropic.com → Settings → API Keys → Create Key.

Jira ve Confluence ayarları arayüzden yapılacak (Bölüm 3.2). Şimdilik sadece ANTHROPIC_API_KEY yeterli.

2.6 Masaüstü ikonu oluştur opsiyonel

5

Çift tıklanabilir Analyst Studio.app üret

Her seferinde terminal açmak yerine masaüstünden başlatmak için:

bash create_app.sh

Bu komut Masaüstü'nde Analyst Studio.app oluşturur. İlk açılışta macOS güvenlik uyarısı verirse:

  1. Sistem Tercihleri → Güvenlik & Gizlilik
  2. "Yine de aç" butonuna tıkla

3. İlk Yapılandırma

3.1 Uygulamayı başlat

İki seçenek:

  • Masaüstü: Analyst Studio.app'i çift tıkla.
  • Terminal: 
    cd ~/projects/brd-analyst-agent
source venv/bin/activate
python app.py

Tarayıcı otomatik olarak http://localhost:5002 adresinde açılır.

3.2 Jira / Confluence bağlantısı opsiyonel

Jira'ya görev oluşturmak veya Confluence'ı RAG bağlamı olarak kullanmak istiyorsanız OAuth bağlantısı gerekir.

1

Atlassian OAuth uygulamasını oluştur (bir kez, yönetici)

developer.atlassian.com/console/myapps → Create → OAuth 2.0 integration → callback URL: http://localhost:5002/api/jira/callback.

Aldığınız Client ID ve Client Secret'i .env dosyasına ekleyin:

JIRA_CLIENT_ID=...
JIRA_CLIENT_SECRET=...
JIRA_PROJECT_KEY=PROJ            # task açılacak proje
JIRA_DEFAULT_SITE=<sitename>.atlassian.net
2

Uygulamadan OAuth ile bağlan

Ayarlar sekmesi → Jira / Confluence Bağlantısı → Bağlan butonu. Tarayıcı Atlassian'a yönlendirir; izinleri onayladıktan sonra geri döner ve token kaydedilir.

OAuth kapsamı (scope) Jira okuma/yazma + Confluence okuma/yazma. Token .env'e şifreli yazılmaz — dosya 0600 izinli tutulur.

3.3 Veri kaynaklarını tanımla

RAG için Confluence space ve Jira projelerini kaydet:

  1. Referanslar sekmesine git
  2. Confluence Spaces bölümüne ilgili space key'lerini ekle (ör. ARCH)
  3. Jira Projects bölümüne proje key'lerini ekle (ör. PROJ)

Bunlar analizlerde otomatik bağlam olarak kullanılır.

3.4 İlk senkronizasyon

1

Veri Kaynakları → Güncelle

Referanslar sekmesi → Güncelle butonu. Confluence sayfaları ve Jira issue'ları yerel reference/ klasörüne çekilir.

İpucu Senkronizasyon dakikalar sürebilir. Tek seferlik yapılır, sonra haftada bir / değişiklik olunca güncellenir.

3.5 UI kodu referansı opsiyonel

Mevcut frontend kodunu RAG bağlamına eklemek isterseniz (yeni ekranların mevcut tasarım diline uyması için):

  1. UI Kodu sekmesi → Zip Yükle
  2. Frontend kaynak klasörünün zip'ini yükle (.tsx, .vue, .css vb.)
  3. Yükleme sonrası teknik analiz Bölüm 16'da mevcut yapıya referans verir

4.2 Akış 1: Süreç → Teknik → Jira

Bu en sık kullanılan akıştır.

1

Dokümanı yükle

Çalıştır sekmesi → Girdi Dokümanı alanına süreç tarifini sürükle-bırak (PDF, DOCX, MD, TXT — tek dosya).

2

Süreç Analizi'ni başlat

Süreç Analizi Çalıştır butonuna bas. AI:

  • Dokümanı okur
  • Senkronize Confluence/Jira/Swagger ile çapraz kontrol yapar
  • output/surec-analizi.md üretir

Süre: ~30-90 sn. İlk çağrıda yavaş, sonraki çağrılar prompt cache ile hızlı.

3

Süreç analizini incele ve onayla

Otomatik olarak Çıktılar sekmesine geçer ve süreç analizini gösterir. Aktörler, adımlar (PA-XXX), iş kuralları (BR-XXX), ekranlar (EK-XXX), açık soruları kontrol et.

  • Eksik/yanlış bir kısım varsa → Yeniden Çalıştır butonu + düzeltme notu yaz
  • Sorun yoksa → Onayla & Teknik Analize Geç
İpucu — Yeniden Çalıştır Tüm dokümanı baştan üretmez; yalnızca belirttiğin bölümü cerrahi olarak günceller. ID'leri ve format korunur.
4

Teknik Analiz

AI süreç analizinden + referanslardan + (varsa) UI kodundan + (varsa) HTML prototipten:

  • DDL (SQL CREATE TABLE)
  • OpenAPI YAML endpoint'leri
  • Validasyon matrisi (FE+BE)
  • Bölüm 16: FE teknik tasarımı (ekran/bileşen/route)
  • Bölüm 17: İş Kırılımı — FE/BE geliştirme görevleri (Jira temeli)

İncele, gerekirse yeniden çalıştır, onayla.

5

HTML Prototip opsiyonel

Onay ekranında HTML Prototip Oluştur butonu. Süreç analizindeki ekranlardan tıklanabilir tek dosyalık mockup.html üretir. Paydaşlarla tasarımı görsel doğrulamak için ideal.

6

Jira görev hiyerarşisi — önizleme + seçim

Çıktılar sekmesinde teknik-analiz.md seçili iken Hiyerarşik Task Oluştur butonuna bas.

Açılan modalda AI'nın önerdiği yapıyı görürsün:

  • 1 Epic + Story'ler + Subtask'lar
  • Her satırda FE veya BE rozeti
  • Checkbox ile hangi öğeleri açacağını seç
  • Başlıkları inline düzenleyebilirsin

Senaryolar:

  • Tek task: sadece Epic'i bırak, diğerlerinin işaretini kaldır → "Devam Et" → 1 issue açılır
  • Tam hiyerarşi: hepsini bırak → bütün ağaç açılır
  • Kısmi: sadece istediklerini seç → seçilenler açılır
Kaskad mantığı Epic'i kapatırsan tüm alt öğeler kapanır. Bir Subtask seçersen parent Story ve Epic otomatik seçilir.

Devam Et'e bastığında modal sana her açılan issue'nun key'ini gösterir (örn. PROJ-123).

4.3 Akış 2: BRD → Kapsam Analizi

Mevcut bir BRD'yi inceletmek veya iki BRD versiyonunu karşılaştırmak için.

1

BRD'yi yükle

Çalıştır sekmesi → BRD dokümanını yükle. Bu akışta üst kısımdaki BRD Analizi butonunu kullan.

2

BRD Analizi sonucu

Üretilenler:

  • brd-analizi.md — fonksiyonel (FR), fonksiyonel olmayan (NFR), user story'ler, kabul kriterleri, tutarsızlıklar
  • brd-sorular.md — Product Owner'a sorulacak en kritik sorular (PO-XXX)
3

BRD'yi referans olarak kaydet

BRD onaylandığında Mevcut BRD olarak kaydet butonu ile dokümanı reference/current-brd/ klasörüne taşı. Bu, ileride kapsam karşılaştırması için baseline olur.

4

Revize BRD ile kapsam karşılaştırması

Zaman içinde BRD'nin yeni versiyonu geldiğinde:

  1. Yeni versiyonu yükle
  2. Kapsam Analizi butonuna bas
  3. Sistem iki versiyonu karşılaştırıp şunları üretir:
    • kapsam-analizi.md — eklenen (YE-XXX), çıkarılan (KL-XXX), değişen (DG-XXX) gereksinimler
    • alternatif-surecler.md — yeni kapsamı karşılayan 3-5 uygulama alternatifi

4.5 Confluence'a yayımlama

Onaylı bir analizi kurumsal wiki'ye yayımlamak için:

  1. Çıktılar sekmesinde yayımlanacak dosyayı seç
  2. Confluence: satırında space seç
  3. Yayımla butonu
  4. Markdown otomatik olarak Confluence Storage Format'a çevrilir, sayfa oluşturulur veya güncellenir

5. İleri Düzey

5.1 Bağlam filtresi

Referans klasörü çok büyüdüyse (yüzlerce Confluence sayfası, binlerce Jira issue), her analizde hepsini AI'a göndermek hem yavaş hem maliyetli olur. Bağlam filtresi ile sadece ilgili olanları dahil edersin.

Ayarlar → Bağlam Filtresi:

  • Anahtar kelimeler: "sipariş, ödeme" → bu kelimeleri içeren dokümanlar gelir
  • Jira Keys: PROJ-123, PROJ-145 → spesifik issue'lar
  • Confluence Pages: sayfa adının bir kısmı → sadece o sayfalar

5.2 Prompt özelleştirme

AI davranışını değiştirmek için 15 sistem promptu arayüzden düzenlenebilir. Ayarlar → Promptlar → ilgili prompt → düzenle → kaydet.

Dikkat Promptlar AI'nın tüm çalışma mantığını belirler. Bozulursa Varsayılana Dön butonu ile sıfırlayabilirsiniz. Önemli değişiklikleri yedekleyin.

5.3 Geçmiş

Her tamamlanmış çalıştırma Geçmiş sekmesinde son 5 kayıtlı tutulur. Eski bir çalıştırmanın çıktılarını görüntüleyebilir, o noktaya geri dönebilirsiniz.

5.4 Uygulamayı güncelle

GitHub'dan son sürümü almak için iki yol vardır:

  • Güncelleme sayfası — uygulamadan tek tıkla git pull --ff-only yapar
  • Terminal: 
    cd ~/projects/brd-analyst-agent
git pull
pip install -r requirements.txt   # gerekirse
Yeniden başlatma Güncelleme sonrası uygulamayı kapatıp açın (Analyst Studio.app'i çık, tekrar aç). Tarayıcı sekmesini yenilemek yeterli değildir; sunucu kodu yenilenmeli.

6. Sorun Giderme

BelirtiOlası neden ve çözüm
Uygulama açılmıyor Terminalden manuel çalıştır: source venv/bin/activate && python app.py. Hata mesajı görünür.
Port 5002 dolu Başka bir terminalden: lsof -i :5002 → eski süreç var → kill PID.
"ANTHROPIC_API_KEY yok" hatası .env dosyasını kontrol et; chmod 600 .env izinli olmalı.
Rate limit (429) hatası Uygulama otomatik tekrar dener; sürekli alırsanız Anthropic plan limitinizi kontrol edin.
Jira "401 Unauthorized" OAuth token süresi dolmuş — Ayarlar → Jira → Yeniden Bağlan.
Confluence "403" hatası OAuth scope eksik. Yeniden bağlanmak gerekir (Confluence scope'ları güncellendiyse).
Cmd+Shift+R sonrası uygulama kapandı Düzeltildi; eski sürüm kullanıyorsanız güncelleyin.
Tarayıcı sekmesini kapattım, uygulama hâlâ açık Heartbeat 45 sn sonra sunucuyu otomatik kapatır.
Çıktı eksik / yanlış Yeniden Çalıştır → düzeltme notu yaz. Sürekli aynı hatayı yapıyorsa promptu (Ayarlar) inceleyin.
Loglar nerede? logs/app.log (rotasyonlu, son 5 dosya, >30 gün eski olanlar otomatik silinir).

7. Sıkça Sorulanlar

Veriler bulutta mı saklanıyor?

Hayır. Tüm dokümanlar ve çıktılar yerel makinanızda kalır. Yalnızca Claude API çağrıları için içerik Anthropic'e gönderilir (Anthropic veri saklama politikası geçerlidir).

Aynı dokümanı tekrar tekrar çalıştırırsam maliyet ne olur?

Prompt caching sayesinde referans bağlam ve sistem promptu önbelleğe alınır; 5 dakika içindeki tekrar çağrılarda token maliyeti ~%90 düşer. Yine de gereksiz tekrar çalıştırmayın.

Birden fazla doküman aynı anda işlenebilir mi?

Şu an her çalıştırma tek doküman üzerinde. Çoklu doküman desteği yol haritasındadır.

Promptu bozdum, nasıl geri alırım?

Ayarlar → Promptlar → ilgili prompt → Varsayılana Dön.

Hangi modelleri kullanıyor?

claude-sonnet-4-6 (analizler), claude-haiku-4-5 (Jira görev başlığı).

Çıktı kalitesi düşükse ne yapayım?

Sırasıyla deneyin:

  1. Girdi dokümanın daha net olduğundan emin olun
  2. İlgili Confluence/Jira/Swagger referanslarının yerel kaynağa senkronlandığından emin olun
  3. Bağlam filtresi ile gürültüyü azaltın
  4. Yeniden Çalıştır + düzeltme notu yazın
  5. Hâlâ yetersizse promptu özelleştirin (Ayarlar)
